# Fish Game — 1단계 구현 지시서 (Stage 1 Build Spec)

> **대상**: 헤르메스 에이전트 (Claude Code 기반 구현)
> **기반 문서**: GAME_DESIGN.md v0.3
> **이 문서의 범위**: 로드맵 1단계 — "코어 3박자(CAST→BITE→REEL) + 미끼 소모형 세션 + 어종 1종"
> **완료 기준**: 브라우저에서 `index.html`을 열면, 미끼가 떨어질 때까지 캐스트→입질→회수를 반복해 물고기를 잡고 점수가 쌓이며, 미끼 소진 시 결과 화면이 뜨고 재시작할 수 있다.

---

## 0. 절대 원칙 (반드시 지킬 것)

1. **라이브러리 0개.** vanilla JS + HTML5 Canvas2D만 사용. npm/빌드 도구 금지.
2. **`file://`로 열어도 동작해야 함.** 외부 fetch 실패 시 폴백 내장 데이터 사용(아래 6-3).
3. **단일 정적 구조.** 이 단계에서는 `index.html` / `style.css` / `game.js` / `data/fish.json` 4파일만.
4. **주석과 함수 분리를 명확히.** 후속 단계(도감·상점·에드센스)에서 확장할 것이므로 상태머신·입력·렌더를 모듈처럼 분리.
5. **매직넘버는 상단 CONFIG 객체로.** 밸런싱 수치를 한곳에서 조정 가능하게.
6. **이 단계에서 만들지 말 것**: 도감, 상점, 업그레이드, 코인 저장, 지역, 시간대, 사운드, 결제, 광고. (전부 후속 단계)

## 2. 상태 머신

이 단계에서 구현할 상태: **`TITLE` → `PLAY` → `RESULT` → (다시) `PLAY`**

```
TITLE  ──[스페이스/클릭]──▶  PLAY
PLAY   ──[미끼 0]────────▶  RESULT
RESULT ──[스페이스/클릭]──▶  PLAY (미끼 리셋, 점수 리셋)
```

- 전역 `state` 변수로 관리. 각 상태별 `update(dt)` / `render(ctx)` 분기.
- 상태 전환은 `changeState(newState)` 단일 함수로. 전환 시 필요한 초기화 수행.

### PLAY 내부의 하위 페이즈 (핵심)

`PLAY` 상태 안에서 3박자를 하위 페이즈로 관리한다. `phase` 변수:

```
IDLE     : 캐스트 대기. 파워 게이지 표시 안 함.
CASTING  : 스페이스 홀드 중. 파워 게이지 0→100→0 왕복(핑퐁).
DRIFT    : 바늘 착지, 입질 대기. 랜덤 시간 후 BITE로.
BITE     : PULL 윈도우 열림(타이밍 바). 시간 내 입력하면 REELING, 놓치면 MISS.
REELING  : 회수 게이지 차오름. 가득 차면 CATCH.
CATCH    : 어획 연출(짧게) → 점수 반영 → IDLE로 복귀.
MISS     : 놓침 연출(짧게) → IDLE로 복귀.
```

페이즈 전이도서:

```
IDLE ──[스페이스 down]──▶ CASTING
CASTING ──[스페이스 up]──▶ (영역 판정)
        ├─ 성공 ─▶ DRIFT (미끼 -1)
        └─ 실패(CAST FAIL) ─▶ IDLE (미끼 -1, "FAIL" 토스트)
DRIFT ──[랜덤 1.5~4.0초 경과]──▶ BITE
BITE ──[윈도우 내 스페이스]──▶ REELING
     └─[윈도우 초과]──▶ MISS
REELING ──[게이지 100%]──▶ CATCH
CATCH ──[0.8초 연출]──▶ IDLE
MISS ──[0.6초 연출]──▶ IDLE
```

> **중요**: 미끼는 **캐스트가 실제 실행되는 순간(CASTING→DRIFT 또는 CAST FAIL 시점)** 1개 소모. IDLE 상태에서 미끼가 0이면 즉시 `RESULT`로 전환.

## 3. 코어 메카닉 상세 (수치 포함)

### 3-1. CAST (투척)
- 입력: **스페이스바 홀드** (마우스 좌클릭 홀드도 동일 처리).
- 홀드 동안 파워 게이지가 0→100→0 **핑퐁**으로 왕복 (속도: `CONFIG.castPingpongSpeed`).
- 릴리스 시점의 파워 값으로 착지 지점 결정.
- **영역 판정**: 파워 `castZone.min`~`castZone.max`(예 35~100) 사이 → 성공. 그 미만 → CAST FAIL.
- **PERFECT 판정**: 파워가 `perfectZone.min`~`perfectZone.max`(예 90~100) → PERFECT 플래그(이 단계에서는 "PERFECT!" 토스트 + 점수 +20만, 희귀확률 로직은 후속).
- 성공/실패 무관하게 **미끼 -1**.

### 3-2. DRIFT / BITE (입질)
- DRIFT 진입 후 `random(driftMin, driftMax)`초(예 1.5~4.0) 대기 → BITE.
- BITE 진입 시 화면 중앙에 **타이밍 바** 표시. 색이 노랑에서 시작해 `biteWindow`초(예 1.2초) 동안 서서히 빨강으로.
- 이 윈도우 안에 스페이스 1회 → REELING. 초과 → MISS.
- (이 단계는 어종 1종이므로 입질 패턴 단일. 대물 예고 신호는 후속 단계.)

### 3-3. REEL (회수)
- REELING 진입 시 회수 게이지 0%.
- 기본 속도 `reelBaseRate`(%/초, 예 40)로 자동 증가.
- **스페이스 홀드 시** `reelBoostRate`(예 70)로 가속 (스태미너 개념은 후속 단계 — 이 단계는 단순 가속만).
- 게이지 100% → CATCH.
- (방해물 로직은 후속 단계. 이 단계는 방해물 없음.)

## 4. 어획 & 스코어링 (1단계 범위)

- 어종은 1종만 사용(`fish.json`의 첫 항목, "고등어").
- CATCH 시: 해당 어종의 점수 범위 내 랜덤값을 `SCORE`에 가산.
- 점수 규칙(이 단계에서 활성):

| 행동 | 점수 |
|------|------|
| CAST 성공(영역 안) | +10 |
| CAST PERFECT | +20 |
| CAST FAIL | 0 (미끼만 소모) |
| BITE MISS | 0 |
| 어획(고등어) | +30 ~ +60 (랜덤) |

- 콤보/희귀/전설/코인은 **이 단계에서 구현하지 않음**(후속). 단, `SCORE` 변수 구조는 후속 확장 가능하게.

## 5. 세션 (출조) 규칙

- PLAY 시작 시 미끼 `CONFIG.baitPerRun`개 지급(예 10).
- 미끼는 캐스트 실행 시점에 1개 소모(성공/FAIL 무관).
- IDLE 상태에서 미끼가 0이 되면 즉시 RESULT로.
- RESULT 화면 표시 내용: 이번 판 SCORE, 잡은 물고기 수, "다음 목표" 자리(이 단계는 placeholder 텍스트 "SHOP/CODEX coming soon"), [한 판 더] 버튼.
- [한 판 더] → PLAY 재시작(SCORE 0, 미끼 리셋).

## 6. 데이터 스키마

### 6-1. `data/fish.json` (배열 구조 — 후속 확장 대비)

```json
{
  "version": 1,
  "fish": [
    {
      "id": "mackerel",
      "name_ko": "고등어",
      "name_en": "Mackerel",
      "rarity": "common",
      "size": "S",
      "region": "east_breakwater",
      "time": "all",
      "scoreMin": 30,
      "scoreMax": 60,
      "coin": 1,
      "reelDifficulty": 1.0,
      "note": "연습용"
    }
  ]
}
```

> 1단계는 `fish[0]`만 사용하되, 나머지 11종을 추가해도 스키마가 깨지지 않게 배열로 설계.

### 6-2. CONFIG 객체 (game.js 상단, 밸런싱용)

```js
const CONFIG = {
  // 세션
  baitPerRun: 10,

  // CAST
  castPingpongSpeed: 140,   // 파워 게이지 왕복 속도 (units/sec)
  castZone: { min: 35, max: 100 },      // 성공 영역
  perfectZone: { min: 90, max: 100 },   // PERFECT 영역

  // DRIFT / BITE
  driftMin: 1.5,            // 입질까지 최소 대기(초)
  driftMax: 4.0,            // 입질까지 최대 대기(초)
  biteWindow: 1.2,          // PULL 윈도우 길이(초)

  // REEL
  reelBaseRate: 40,         // 자동 회수 속도(%/sec)
  reelBoostRate: 70,        // 홀드 시 회수 속도(%/sec)

  // 연출
  catchAnimTime: 0.8,       // CATCH 연출(초)
  missAnimTime: 0.6,        // MISS 연출(초)

  // 점수
  score: { castOk: 10, perfect: 20 }
};
```

### 6-3. 폴백 데이터 (file:// 대응)
`fetch('data/fish.json')` 실패 시 `game.js` 안에 하드코딩된 동일 구조의 폴백 객체를 사용. **fetch 실패로 게임이 멈추면 안 됨.**

## 7. 게임 루프

- `requestAnimationFrame` 기반, **고정 timestep**(dt = 1/60초). 누적 시간 방식(accumulator)으로 프레임 독립적 업데이트.
- 매 프레임: `update(dt)` → `render(ctx)`.
- Canvas 크기: 논리 해상도 예 960×540, CSS로 반응형 스케일(비율 유지).

---

## 8. 입력 (Input 어댑터)

- 단일 `Input` 객체로 관리(후속 터치 확장 대비).
- 키보드: `Space`(주 액션), `Enter`(메뉴 확인).
- 마우스: 캔버스 클릭 = Space와 동일 취급, 버튼 클릭은 별도.
- **홀드/탭 구분 필요**: CAST(홀드), PULL(탭), REEL 부스트(홀드). `keydown`/`keyup`으로 홀드 상태 추적.
- 브라우저 스크롤 방지: 스페이스 기본 동작 `preventDefault`.

## 9. UI / HUD (1단계)

Canvas 위 또는 오버레이 DOM으로 표시:

- 좌상단: **SCORE**
- 우상단: **미끼 남은 수** (🪱 ×N)
- 하단 중앙: **CAST 파워 게이지**(CASTING 페이즈에서만)
- 중앙: **입질 타이밍 바**(BITE 페이즈), **회수 게이지**(REELING 페이즈)
- 토스트(짧게 페이드): "PERFECT!", "FAIL", "MISS", "+45" 등
- 바다/배는 간단한 도형·색으로(파스텔 톤, 팔레트: 바다 #4FB3D9→#2B7A9B, 포인트 #E76F51). 정교한 일러스트 불필요.

### 화면 참고

│                                            │
│              🪝                            │

- **PLAY**: 위 HUD + 페이즈별 게이지.

## 11. 완료 기준 (Acceptance Criteria)

이 단계는 아래가 **모두** 충족되면 완료:

1. `index.html`을 브라우저(및 `file://`)에서 열면 TITLE 화면이 뜬다.
2. 스페이스로 PLAY 시작, 미끼 10개가 표시된다.
3. 스페이스 홀드→릴리스로 캐스트되고, 파워가 영역 밖이면 "FAIL" + 미끼 -1.
4. 캐스트 성공 시 일정 시간 후 입질 타이밍 바가 뜬다.
5. 윈도우 내 스페이스로 회수가 시작되고, 게이지가 차면 물고기를 잡아 점수가 오른다("+45" 등).
6. 윈도우를 놓치면 "MISS", 물고기는 못 잡지만 벌점 없음.
7. 미끼가 0이 되면 RESULT 모달이 뜨고 SCORE·잡은 수가 표시된다.
8. [한 판 더]로 SCORE·미끼가 리셋되어 재시작된다.
9. `CONFIG`의 수치만 바꿔도 밸런스가 바뀐다(하드코딩된 매직넘버 없음).
10. 콘솔 에러 없이 동작한다.
